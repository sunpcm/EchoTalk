# EchoTalk 全局开发规则与上下文指令

你好，Claude Code。在本项目的所有开发中，请你严格遵守以下原则：

## 1. 语言、文档与交付规范

- **绝对中文交互**：对话、代码注释 (JSDoc/Docstrings) 以及所有的说明文档，必须使用规范的中文。
- **阶段性交付物**：在每个 Phase 开发完成后，你必须主动生成或更新两份技术文档：
  1. `docs/PHASE_X_TECH_STACK.md`：说明本阶段核心逻辑与技术栈。
  2. `docs/PHASE_X_MANUAL_TEST.md`：提供人工验证指南。
- **持续更新 README (重点)**：在每个 Phase 的最后一步，你必须更新项目根目录下的 `README.md` 文件。
  - 更新“项目当前状态与进度”。
  - 更新“如何启动当前项目”（包括新增的后端 worker、数据库或前端命令）。
  - 更新“当前已支持的功能特性”。

## 2. 架构与 Monorepo 规范

- **Monorepo 结构**：本项目采用 Monorepo。前端为 `TypeScript + React (Next.js 15)`，后端为纯 `Python + FastAPI`。
- **保护现有资产**：项目中**已经实现了 Login 和用户管理模块**。绝对不要重新去写注册、登录页面或 Auth 基础逻辑！每次写代码前，先确认存放目录。
- **国际化 (i18n) 第一优先**：系统必须预留多语言能力。
  - 前端：任何硬编码的中文文案，必须通过 i18n 框架（如 `next-intl` 或 `react-i18next`）的 key 来映射。现阶段只需维护一份中文语言包（如 `zh-CN.json`），但代码结构必须是支持多语言的。
  - 后端：返回的用户提示信息（如报错、业务提示）也应具备错误码或 i18n 映射机制。

## 3. Auth 剥离与 Mocking 机制

- **全局 Mock 拦截**：由于当前开发阶段绕过真实的 Auth 系统，请在后端的 FastAPI 依赖（Depends）中，实现一个全局的 `get_current_user` Mock 函数，直接返回写死的测试用户（如 `{"id": "test-user-uuid-1234", "email": "test@example.com"}`）。
- **TODO 标记**：在 Mock 函数处明确标记 `// TODO: 后续接入真实 JWT 校验逻辑`。

## 4. 代码质量与验证底线

- **无错误提交**：你生成的代码必须保证能够通过项目现有的代码质量检查，绝不能在 commit 时被 Husky 拦截。
- **自动检查**：在告诉我“代码编写完成”之前，你必须自己在终端中运行相应的检查命令（例如前端的 `pnpm lint`, `tsc --noEmit`，后端的 `flake8`, `mypy` 等）。
- **测试代码豁免**：现阶段可以暂时不编写单元测试代码（业务逻辑的 unit tests），但保证核心链路可通过 curl 或手动验证。

## 5. 执行流程

- 每次写代码前，先陈述你的实现思路。
- 给出代码后，自行在终端运行编译、Lint 校验和 curl 接口自测。
