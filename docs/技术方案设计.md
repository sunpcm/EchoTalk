| `@livekit/components-react` | WebRTC 语音交互       |

### 3.2 后端技术栈

| 技术                          | 用途                    |
| ----------------------------- | ----------------------- |
| Python + FastAPI              | API 服务器              |
| SQLAlchemy + Alembic          | ORM + 数据库迁移        |
| PostgreSQL                    | 主数据库                |
| Redis                         | 会话缓存、Celery broker |
| Celery                        | 异步任务队列            |
| ChromaDB (本地) / Qdrant (云) | 向量数据库（RAG）       |
| LiveKit Python SDK            | WebRTC 后端代理         |

### 3.3 AI/API 服务

| 服务                 | 场景                             | API                             |
| -------------------- | -------------------------------- | ------------------------------- |
| **OpenRouter**       | LLM 统一调用（支持切换各种模型） | `https://openrouter.ai/api/v1`  |
| **硅基流动**         | 国内用户/中文优化模型备选        | `https://api.siliconflow.cn/v1` |
| **Deepgram Flux**    | 实时 STT（含回合结束检测）       | Deepgram API                    |
| **Cartesia Sonic 3** | TTS（40-90ms 超低延迟）          | Cartesia API                    |
| **ELSA Speech API**  | 专业发音评估（Pro+）             | ELSA API                        |
| **Hume EVI 3**       | 情绪感知 S2S 对话（Premium）     | Hume API                        |
| **Sync.so**          | 数字人唇形同步（基础版）         | Sync.so API                     |
| **LiveKit**          | WebRTC 媒体路由基础设施          | LiveKit Cloud                   |

### 4.2 对话 System Prompt 模板（教练角色）

LLM 扮演口语教练的 Prompt 需要包含：

- 角色设定（友善、有耐心的 AI 英语教练）
- 当前场景（如：面试模拟、日常对话、雅思备考）
- 用户当前薄弱技能标签（来自 BKT 输出）
- 错误纠正策略：**隐性重述（Recast）**，不打断用户，在自然对话中用正确形式重复
- 情绪状态指令（来自情绪分析）：焦虑时降低复杂度，鼓励为主

## 5. 实时对话管线

### 5.1 免费版：优化级联架构

```
用户说话（WebRTC 音频流）
    │
    ▼
LiveKit Agent 接收音频
    │
    ├──→ [主通道 - 低延迟] Deepgram Flux STT（流式）
    │         │
    │         ▼
    │    OpenRouter LLM（轻量快速模型，流式输出）
    │         │
    │         ▼
    │    Cartesia TTS（状态空间模型，40-90ms 首包）
    │         │
    │         ▼
    │    WebRTC 回传用户
    │
    └──→ [异步通道 - 精准分析] Whisper Large（慢速精准）
              │
              ▼
         发音分析 + 语法分析 → Celery任务 → DB → 更新知识状态

目标延迟：< 800ms 端到端
```

### 5.2 Premium 版：情绪感知 S2S

```
用户说话（WebRTC 音频流）
    │
    ▼
LiveKit Agent 接收音频
    │
    ├──→ [主通道] Hume EVI 3 API（原生 S2S）
    │         │ 情绪感知语音直接生成响应音频
    │         ▼
    │    WebRTC 回传用户
    │
    └──→ [异步通道] 音频特征提取 → 情绪状态记录 → 知识状态更新

特殊场景（多模态，如图片描述题）→ 切换 Gemini Multimodal Live
```

## 8. RAG 知识检索流程

```
用户技能状态（BKT 输出弱项）
    │
    ▼
构建检索 Query：
  "scenario: business_meeting, weak_skills: [θ音发音, 过去完成时], difficulty: B1→B2"
    │
    ▼
向量检索 ChromaDB（语料库：精选对话、语法解释、发音指南）
    │
    ▼
候选练习材料 Top-K（按 Krashen i+1 难度过滤）
    │
    ▼
LLM 生成围绕该素材的对话场景指令（System Prompt 注入）
    │
    ▼
进入当次对话会话
```

## 10. 数据库设计（PostgreSQL）

```sql
-- 用户表
users (
    id UUID PRIMARY KEY,
    email VARCHAR UNIQUE,
    password_hash VARCHAR,
    subscription_tier ENUM('free', 'pro', 'premium'),
    created_at TIMESTAMP
)

-- 用户配置
user_profiles (
    user_id UUID REFERENCES users,
    native_language VARCHAR,        -- e.g. 'zh-CN'
    target_level VARCHAR,           -- e.g. 'B2', 'IELTS 7'
    learning_goals TEXT[],          -- e.g. ['job_interview', 'daily_chat']
    total_practice_minutes INT DEFAULT 0
)

-- 练习会话
sessions (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users,
    mode ENUM('pronunciation', 'free_talk', 'scenario', 'exam_prep'),
    scenario_id INT REFERENCES scenarios,
    started_at TIMESTAMP,
    ended_at TIMESTAMP,
    tier_used ENUM('free', 'pro', 'premium')  -- 本次用的哪个管线
)

-- 会话转录
transcripts (
    id BIGSERIAL PRIMARY KEY,
    session_id UUID REFERENCES sessions,
    role ENUM('user', 'assistant'),
    content TEXT,
    audio_url VARCHAR,              -- S3/R2 存储路径
    timestamp_ms BIGINT
)

-- 发音评估结果
pronunciation_assessments (
    id UUID PRIMARY KEY,
    session_id UUID REFERENCES sessions,
    transcript_id BIGINT REFERENCES transcripts,
    phoneme_alignment JSONB,        -- [{phoneme, expected, got, type, position}]
    overall_score FLOAT,
    fluency_score FLOAT,
    completeness_score FLOAT,
    elsa_response JSONB             -- 原始 ELSA API 返回（Pro+）
)

-- 语法错误记录（Celery 异步写入）
grammar_errors (
    id UUID PRIMARY KEY,
    session_id UUID REFERENCES sessions,
    error_type VARCHAR,             -- e.g. 'tense_error', 'subject_verb_agreement'
    original TEXT,
    corrected TEXT,
    skill_tag VARCHAR               -- 关联到 skills 表
)

-- 知识状态（BKT 结果）
knowledge_states (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users,
    skill_id INT REFERENCES skills,
    p_mastery FLOAT DEFAULT 0.1,    -- 当前掌握概率
    attempt_count INT DEFAULT 0,
    last_practiced_at TIMESTAMP,
    UNIQUE(user_id, skill_id)
)

-- 技能定义
skills (
    id SERIAL PRIMARY KEY,
    category VARCHAR,               -- 'phonetics', 'grammar', 'vocabulary', 'fluency'
    name VARCHAR,                   -- e.g. 'phoneme_θ', 'past_perfect_tense'
    display_name VARCHAR,
    difficulty_level FLOAT          -- 1.0-10.0
)

-- 场景配置
scenarios (
    id SERIAL PRIMARY KEY,
    name VARCHAR,
    description TEXT,
    required_level VARCHAR,
    skill_focus TEXT[],             -- 该场景主要锻炼的技能
    system_prompt_template TEXT     -- LLM System Prompt 模板
)
```

#### BKT 更新公式参考

```python
# 给定一次练习结果（correct=True/False），更新掌握概率
def update_mastery(p_mastery, correct, params):
    if correct:
        likelihood = p_mastery * (1 - params.p_slip) + (1 - p_mastery) * params.p_guess
        p_posterior = p_mastery * (1 - params.p_slip) / likelihood
    else:
        likelihood = p_mastery * params.p_slip + (1 - p_mastery) * (1 - params.p_guess)
        p_posterior = p_mastery * params.p_slip / likelihood
    # 学习发生
    return p_posterior + (1 - p_posterior) * params.p_transit
```
