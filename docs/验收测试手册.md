# EchoTalk — 验收测试手册

> 覆盖 Phase 1（基础对话管线）和 Phase 2（发音评估 + 知识追踪）。
> 所有 curl 示例假设后端运行在 `http://localhost:8000`，前端运行在 `http://localhost:3000`。

---

## 一、环境准备

### 1.1 启动后端

```bash
cd backend
source .venv/bin/activate
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

健康检查：

```bash
curl http://localhost:8000/api/health
# 期望: {"status":"ok","service":"echo-talk"}
```

```bash
cd frontend
pnpm dev
```

访问 `http://localhost:3000`，应跳转至登录页。

## 二、Phase 1 验收：基础对话管线

### 目标

用户可注册/登录、创建会话、发送消息并收到 AI 回复（含 TTS 音频 base64）。

---

### 2.1 用户认证

#### 注册

```bash
curl -s -X POST http://localhost:8000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"pass1234","name":"Test User"}' \
  | python3 -m json.tool
```

**期望响应**：

```json
{
  "user": {
    "id": "<uuid>",
    "email": "test@example.com",
    "name": "Test User",
    "created_at": "<iso8601>"
  },
  "access_token": "<jwt>",
  "token_type": "bearer"
}
```

**验收**：HTTP 200，`access_token` 非空。

---

#### 登录

```bash
TOKEN=$(curl -s -X POST http://localhost:8000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"pass1234"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")
echo "TOKEN=$TOKEN"
```

**验收**：`TOKEN` 输出为有效 JWT 字符串。

---

#### 查询当前用户

```bash
curl -s http://localhost:8000/api/auth/me \
  -H "Authorization: Bearer $TOKEN" \
  | python3 -m json.tool
```

**验收**：返回与注册时相同的用户信息，HTTP 200。


### 2.3 对话（核心链路）

#### 发送消息

```bash
curl -s -X POST http://localhost:8000/api/conversation/chat \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"session_id\":\"$SESSION_ID\",\"message\":\"Hello, can you help me practice English?\"}" \
  | python3 -m json.tool
```

**期望响应**：

```json
{
  "reply": "<AI 回复文本>",
  "audio_base64": "<base64 字符串或 null>",
  "transcript_id": "<uuid>"
}
```

**验收**：

- `reply` 非空字符串（mock 模式下为预设回复）
- `transcript_id` 为合法 UUID
- HTTP 200

---

#### 多轮对话

```bash

curl -s -X POST http://localhost:8000/api/conversation/chat \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"session_id\":\"$SESSION_ID\",\"message\":\"I want to improve my pronunciation.\"}" \
  | python3 -c "import sys,json; d=json.load(sys.stdin); print('reply:', d['reply'][:80])"
```

**验收**：AI 回复内容合理，HTTP 200。

---

#### 查询会话（含转录记录）

```bash

```

**验收**：

- `transcripts` 数组中存在 `role: "user"` 和 `role: "assistant"` 条目

### 3.3 Needleman-Wunsch 对齐单元测试

```bash
cd backend
.venv/bin/python3 - <<'EOF'
from services.pronunciation.phoneme_aligner import align_phonemes, compute_pronunciation_score

# 完全匹配
alignment = align_phonemes(["AH", "N", "D"], ["AH", "N", "D"])
assert all(a["type"] == "correct" for a in alignment), "All should be correct"
score = compute_pronunciation_score(alignment)
assert score == 100.0, f"Score should be 100, got {score}"
print("Perfect match: OK")

# 替换错误
alignment = align_phonemes(["TH", "IH", "N"], ["S", "IH", "N"])
subs = [a for a in alignment if a["type"] == "substitution"]
assert len(subs) == 1, f"Expected 1 substitution, got {len(subs)}"
assert subs[0]["expected"] == "TH" and subs[0]["actual"] == "S"
print("Substitution detection: OK")

# 缺失（deletion）
alignment = align_phonemes(["AH", "N", "D"], ["AH", "D"])
dels = [a for a in alignment if a["type"] == "deletion"]
assert len(dels) == 1
print("Deletion detection: OK")

# 多余（insertion）
alignment = align_phonemes(["AH", "N"], ["AH", "HH", "N"])
ins = [a for a in alignment if a["type"] == "insertion"]
assert len(ins) == 1
print("Insertion detection: OK")

print("All NW alignment tests passed.")
EOF
```

**验收**：输出 4 行 `OK` 和最终 `All NW alignment tests passed.`

## 四、Mock 模式说明

当前所有外部依赖均处于 Mock 模式（`backend/config.py` 中 `USE_MOCK_*=True`），行为如下：

| 服务         | Mock 行为                                  | 接入真实服务所需                                                      |
| ------------ | ------------------------------------------ | --------------------------------------------------------------------- |
| LLM          | 返回固定回复文本                           | 设 `USE_MOCK_LLM=False`，填写 `OPENROUTER_API_KEY`                    |
| TTS          | 返回 `None`（无音频）                      | 设 `USE_MOCK_TTS=False`，填写 `CARTESIA_API_KEY`                      |
| STT          | 不使用（对话通过 REST `message` 字段传入） | LiveKit + Deepgram 接入后自动流式转录                                 |
| LiveKit      | 不使用                                     | 设 `USE_MOCK_LIVEKIT=False`，填写三个 `LIVEKIT_*` 变量                |
| Celery       | `.delay()` 同步执行，无需 Redis            | 安装 `celery redis`，设 `USE_MOCK_CELERY=False`，启动 `celery worker` |
| ELSA         | 返回随机发音分数                           | 设 `USE_MOCK_ELSA=False`，填写 `ELSA_API_KEY`                         |
| phoneme 查询 | letter-by-letter fallback                  | 安装 `pip install pronouncing` 后自动启用 CMU dict                    |

## 五、一键集成测试脚本

将以下内容保存为 `backend/integration_test.sh` 并执行：

```bash
#!/usr/bin/env bash
# Phase 1 + Phase 2 快速集成验证
set -e

BASE="http://localhost:8000/api"

echo "=== [1/7] Health check ==="
curl -sf $BASE/health | python3 -c "import sys,json; d=json.load(sys.stdin); assert d['status']=='ok'"
echo "OK"

echo "=== [2/7] Register ==="
TOKEN=$(curl -sf -X POST $BASE/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"ci_'$RANDOM'@test.com","password":"pass1234","name":"CI User"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")
echo "OK (token obtained)"

echo "=== [3/7] Create session ==="
SID=$(curl -sf -X POST $BASE/sessions \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"mode":"conversation"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['id'])")
echo "OK (session=$SID)"

echo "=== [4/7] Chat (including a grammar error) ==="
curl -sf -X POST $BASE/conversation/chat \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"session_id\":\"$SID\",\"message\":\"Yesterday I go to school and I think about this thing.\"}" > /dev/null
echo "OK"

echo "=== [5/7] End session (triggers analysis) ==="
curl -sf -X POST $BASE/sessions/$SID/end \
  -H "Authorization: Bearer $TOKEN" > /dev/null
echo "OK"

echo "=== [6/7] Check assessment ==="
SCORE=$(curl -sf $BASE/assessments/$SID \
  -H "Authorization: Bearer $TOKEN" \
  | python3 -c "import sys,json; d=json.load(sys.stdin); assert 0<=d['overall_score']<=100; print(d['overall_score'])")
echo "OK (score=$SCORE)"

echo "=== [7/7] Check knowledge states ==="
COUNT=$(curl -sf $BASE/assessments/knowledge/states \
  -H "Authorization: Bearer $TOKEN" \
  | python3 -c "import sys,json; lst=json.load(sys.stdin); assert len(lst)>0; print(len(lst))")
echo "OK ($COUNT skill states updated)"

echo ""
echo "All Phase 1 + Phase 2 integration tests passed."
```

运行方式：

```bash
chmod +x backend/integration_test.sh
bash backend/integration_test.sh
```

**验收**：最后一行输出 `All Phase 1 + Phase 2 integration tests passed.`
